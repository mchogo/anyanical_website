# UI / UX Guidelines

## 基本方針

金融ダッシュボードとして、派手なランディングページよりも「週末に相場差をすばやく確認できる」実用性を優先します。

優先順位:

1. 価格、変化率、データ状態がすぐ読める
2. 週末モードの意味と注意点が誤解なく伝わる
3. 関連ツールへ少ない操作で移動できる
4. スマホでも固定ナビやカードが本文を邪魔しない

## レイアウト

- 初回アクセスはHOMEを第一表示に置き、左上ブランドからHOMEへ戻れるようにする
- 相場ボードは固定ナビの「相場ボード」から開ける主要ツールとして扱う
- チャート、説明、FAQ、関連ツールは下部へ自然に続ける
- 戦略、コミュニティ、プレミアムの情報ページは、外部リンクへ急がせず、確認事項とリスク説明を同じ画面内に置く
- サブスク、コピートレード、EA案内は目的を分けて表示し、利益保証や過度な期待を想起させる表現を避ける
- プレミアムページは加入手順より先に、継続率、日々の相場考察、限定チャンネル、限定ツールなどの利用価値を見せる
- 上部固定ナビと重複する全ページ一覧は本文内に置かず、ページ末尾に文脈別の次に見るものだけを置く
- ページセクションは大きな余白と区切りで整理する
- カードの角丸は控えめにし、金融ツールとして落ち着いた見た目にする

## 固定ナビ

`FloatingNav.tsx` は画面上部に追尾します。

注意:

- 本文冒頭が隠れないよう `App.tsx` 側で上余白を確保する
- モバイルでは横並びの全リンクを出さず、ブランド名とメニューボタンを1行に収める
- 開いたメニューは2列グリッドを基本にし、縦に伸びても画面内スクロールできるようにする
- 外部リンクと内部リンクの違いが分かる表現にする

## 相場カード

表示するべき情報:

- 銘柄名
- 現在価格
- 金曜基準からの変動率
- 過去数時間の価格形状
- データソースまたは状態
- 公式市場との関係を示す補足

状態:

- 接続中
- 更新済み
- データなし
- 比較価格なし

## チャート

- 初期表示は需要の高い金・日経・ドル円・BTCを優先する
- TradingView埋め込みが表示できない場合に備え、ユーザーが別タブや別銘柄へ切り替えられる構成にする
- 先物シンボルで表示制限が出る場合は、OANDAなどのCFD/FXシンボルを優先する
- 経済指標カレンダーは外部ウィジェット内の文字色崩れを避けるため、ライトテーマの独立パネルとして表示する

## 文言

推奨:

- 「参考値」
- 「24時間取引価格」
- 「週末参考価格」
- 「公式市場休場中の目安」
- 「金曜基準」

避ける:

- 「公式価格」
- 「確定価格」
- 「Hyperliquid上のperp価格」
- 「必ず」
- 「保証」
- 「勝てる」

## レスポンシブ確認

UI変更時は最低限以下を確認します。

- 375px幅で価格カードが横にはみ出さない
- 固定ナビがヘッダーやページタイトルを隠さない
- iframeやTradingView widgetが親幅を超えない
- ボタン内テキストが潰れない
